## 🚀 Next Steps to Complete Deployment

### Step 1: Get PostgreSQL Password
The database password was auto-generated by Terraform. Get it from:

```bash
# Option 1: From Terraform output (if state is accessible)
cd terraform/environments/dev
terraform output postgres_password

# Option 2: Reset it via Azure CLI
az postgres flexible-server update \
  --resource-group ari-rg-capstone-dev \
  --name psql-capstone-dev-ari999 \
  --admin-password "YourNewSecurePassword123!"
```

### Step 2: Update Kubernetes Secret
Edit `capstone-api/k8s-azure/02-secrets.yaml`:
```yaml
stringData:
  POSTGRES_USER: "psqladmin"
  POSTGRES_PASSWORD: "ACTUAL_PASSWORD_HERE"  # Replace this
```

### Step 3: Initialize Database Schema
Connect to PostgreSQL and create the table:

```bash
# Connect to PostgreSQL
psql "host=psql-capstone-dev-ari999.postgres.database.azure.com \
      port=5432 \
      dbname=capstone_db \
      user=psqladmin"

# Run the schema
CREATE TABLE IF NOT EXISTS requests (
  id SERIAL PRIMARY KEY,
  backend_name TEXT NOT NULL,
  ts TIMESTAMP DEFAULT NOW(),
  meta JSONB,
  image BYTEA
);
```

### Step 4: Deploy Backends
1. Commit all changes
2. Push to repository
3. Create new pipeline in Azure DevOps using `azure-backend-pipeline.yml`
4. Run the pipeline

**API B:**
